InP

Interactive Pages — macht Server-Side Rendering dynamisch. Links und Buttons öffnen Inhalte als Modal oder aktualisieren Teile der Seite, ohne Seiten-Reload und ohne Framework.

Modal · Einfach

Ein Link oder Button öffnet die Zielseite als Dialog — ein Attribut genügt.

<a href="info.html"
   data-inp-typ="modal">
  Öffnen
</a>
open_in_new Demo
Modal · data-extract

Hat die Zieldatei Header/Footer, lädt data-extract nur den gewünschten Bereich.

<button
  data-inp-typ="modal"
  data-href="login.html"
  data-extract="#form">
  Login
</button>
Submit · data-target

Nach einem Formular-Submit im Modal wird nur der per data-target angegebene Bereich der Hauptseite ausgetauscht.

<a href="edit.php"
   data-inp-typ="modal"
   data-target="#list">
  Bearbeiten
</a>

01 — Quickstart

Einbinden

InP als ES-Modul im <head> einbinden. Danach reicht es, die data-inp-*-Attribute auf vorhandene Elemente zu setzen — keine weitere Initialisierung nötig.

<script src="inp_v1_0_0.js" type="module"></script>

Modal öffnen

<!-- Link -->
<a href="seite.html" data-inp-typ="modal">Öffnen</a>

<!-- Button (data-href statt href) -->
<button data-inp-typ="modal" data-href="seite.html">Öffnen</button>

Modal mit Formular und Update

<!-- Hauptseite -->
<ul id="user-list"> ... </ul>

<a href="add-user.php"
   data-inp-typ="modal"
   data-target="#user-list">Hinzufügen</a>

<!-- add-user.php: Formular im Modal -->
<form action="save.php" method="post">
  <input type="text" name="name">
  <a data-inp-typ="close-btn">Abbrechen</a>
  <button data-inp-typ="submit-btn">Speichern</button>
</form>

<!-- save.php gibt #user-list zurück → InP tauscht es auf der Hauptseite aus -->
02 — Attribute

Attribute

data-inp-typ

Pflichtattribut auf jedem InP-Element.

WertBeschreibungVoraussetzung
modal Lädt die Zielseite und öffnet sie als nativen <dialog>. CSS und JS der Zieldatei werden isoliert ausgeführt. href oder data-href
update Lädt die Zielseite und ersetzt das in data-target angegebene Element direkt auf der Seite. data-target
submit-btn Sendet das umgebende <form> per Fetch ab, aktualisiert optional data-target und schließt den Dialog. muss in einem <form> liegen
close-btn Schließt den umgebenden Dialog ohne weitere Aktion. muss in einem <dialog> liegen

data-href

Alternative zu href für Elemente ohne <a>-Tag. InP prüft zuerst data-href, dann href.

data-extract

CSS-Selektor, der festlegt welcher Teil der Zielseite in den Modal geladen wird. Ohne dieses Attribut wird der gesamte <body> verwendet.

data-target

CSS-Selektor, der festlegt welches Element auf der aktuellen Seite nach Submit oder Update ersetzt wird. Existiert das Element in der Server-Antwort, wird es dort herausgeschnitten und eingefügt. Ist kein Target gesetzt und ein Redirect erkannt, navigiert die Hauptseite zur Ziel-URL.

03 — Modal

Modal-Styling

Der erzeugte <dialog> trägt die Klasse inp-modal und bringt Standard-Styles mit. Überschreibbar per CSS-Variablen:

:root {
  --inp-bg:       #ffffff;
  --inp-text:     #1a1a1a;
  --inp-border:   #e0e0e0;
  --inp-shadow:   0 10px 25px rgba(0,0,0,.1);
  --inp-backdrop: rgba(0,0,0,.3);
}

Dark Mode wird automatisch via @media (prefers-color-scheme: dark) unterstützt.

JS-Isolation

Inline-Skripte der Zieldatei werden so ausgeführt, dass document-Aufrufe wie querySelector und getElementById nur Elemente innerhalb des Modals betreffen — nicht die Hauptseite. ES-Module (type="module") werden dabei über einen Proxy korrekt behandelt.

Redirect-Erkennung

Nach einem Submit erkennt InP Redirects automatisch — via PHP header(), Meta-Refresh oder JavaScript location.href. Ist ein data-target gesetzt, wird die Redirect-Zielseite geladen und nur das Zielelement ausgetauscht. Ohne Target: Hauptseite navigiert zur Ziel-URL.

info

InP ist für Server-Side Rendering gedacht. Die Modal- und Submit-Demos benötigen einen lokalen Webserver oder ein PHP-Backend.
