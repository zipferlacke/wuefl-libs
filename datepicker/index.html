01 — Installation

Einbinden

Als ES-Modul importieren. datepicker.css muss im selben Verzeichnis liegen und wird automatisch injiziert.

import { DatePicker } from './datepicker.js';

Single-Datum

const dp = new DatePicker({ lang: 'de' });
dp.create(document.getElementById('meinInput'));

document.getElementById('meinInput')
  .addEventListener('change', e => console.log(e.target.value));

Zeitspanne (Range)

// Zwei Inputs als Array → automatisch Range-Modus
dp.create([
  document.getElementById('von'),
  document.getElementById('bis'),
]);

Datum + Uhrzeit

dp.create(inputEl, { showTime: true });

Nur Zeit

dp.create(inputEl, { showDate: false, showTime: true });

Ausgabeformat wählen

// 'locale' (default) → "13.03.2026" / "13.03.2026, 14:30"
// 'ms'               → Unix-Millisekunden: "1742169600000"
// 'iso'              → "2026-03-13" / "2026-03-13T14:30"
dp.create(inputEl, { outputFormat: 'ms' });
02 — API

Konfiguration

new DatePicker(config)

ParameterTypBeschreibung
config.langstringSprache der UI: 'de' oder 'en'. Standard: Browser-Sprache.
config.localestringLocale für Datumsformatierung, z.B. 'de-DE'. Standard: navigator.language.
config.translationsObjectEigene Texte für Buttons und Labels.
config.optionsObjectGlobale Standardoptionen für alle Picker dieser Instanz.

dp.create(inputConfig, options)

inputConfigErgebnis
elementSingle-Modus
[element]Single-Modus
[start, end]Range-Modus — automatisch erkannt
OptionTypStandardBeschreibung
showDatebooleantrueKalender anzeigen.
showTimebooleantrueZeitpicker anzeigen.
outputFormat'locale'|'ms'|'iso''ms'Format des gespeicherten Werts. Trigger-Anzeige ist immer Regionalformat.

Instanz-API

MethodeBeschreibung
instance.open()Picker programmatisch öffnen.
instance.close()Schließen ohne Speichern.
instance.save()Speichern und schließen.
instance.reset()Auswahl zurücksetzen.

Verhalten

Single ohne Zeit: Klick auf Tag → sofort speichern & schließen.

Single mit Zeit: Speichern-Button erscheint — erst nach Klick wird geschlossen.

Range: Erster Klick = Von, zweiter Klick = Bis. Hover zeigt Vorschau. Speichern-Button immer vorhanden.

Range-Konvention: Nach dem ersten Klick ist Von = Bis. Der zweite Klick weitet den Bereich aus.

03 — CSS-Variablen

Anpassen

Alle Variablen einmal in :root setzen. Lightness-Abstufungen werden automatisch berechnet — nur der Farbton (h + s) der Basis-Farbe ist relevant.

VariableBeschreibung
--clr-pickerFarbton des Popovers (h + s)
--clr-picker-triggerFarbton des Trigger-Elements
--clr-picker-inputAkzentfarbe: Buttons, Auswahl, Range-Balken
--clr-picker-todayFarbe des Heute-Kreises (Standard: Orange)
--fs-picker-headerSchriftgröße Header/Monat
--fs-input-pickerSchriftgröße Tage & Inputs
--br-pickerBorder-Radius
--height-input-pickerHöhe Buttons & Zeit-Inputs
--picker-cal-widthBreite des Kalenders (Standard: 290px)
--picker-sel-sizeDurchmesser des Auswahl-/Heute-Kreises

Versionierung

Die Dateien datepicker.js und datepicker.css sind Loader-Wrapper, die immer auf die aktuelle Version zeigen. Beim Release nur eine Zeile im Loader ändern — alle HTML-Dateien bleiben unberührt.

// datepicker.js — beim Release nur diese Zeile ändern:
export { DatePicker } from './datepicker_v1.0.1.js';
